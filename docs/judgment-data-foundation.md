## 질문 1. 의사결정의 옳고 그름은 어떻게 판단하는가

### 핵심 문제: "결과로 판단하기"의 함정

Annie Duke가 "resulting"이라 부른 인지 오류 — 결과가 좋으면 좋은 결정, 나쁘면 나쁜 결정으로 판단하는 것. 이건 인간의 기본 인지 모드이지만, 구조적으로 틀렸다.

```
                좋은 결과        나쁜 결과
좋은 결정      당연한 성공       불운
나쁜 결정      운 좋은 성공     당연한 실패
```

인간은 이 2x2를 자연스럽게 1x1로 축소한다 ("좋은 결과 = 좋은 결정").
이 문제를 진지하게 다루는 모든 분야가 이에 대한 방어 메커니즘을 개발했다.

### 각 분야가 실제로 쓰는 방법

**포커 — 가장 깨끗한 사례**

- 의사결정을 **기대값(EV)**으로 평가한다. 80% 승률로 올인했다가 졌다면, 그건 *좋은 결정*이다.
- 결정 품질이 결과로 수렴하려면 **수천 핸드**가 필요하다. 개별 핸드는 무의미.
- 결과를 보기 *전에* 추론 과정을 기록한다 (PokerTracker 등).

교훈: 같은 유형의 결정이 수천 번 반복될 때만 통계적 수렴이 가능하다. 전략적 의사결정은 이 조건을 거의 충족하지 못한다.

**투자 — Bridgewater**

- **사전 기록**: 투자 전에 논지, 확률 분포 추정, 핵심 가정, 무효화 조건을 기록. 타임스탬프 + 변경 불가.
- **"야구 카드"**: 각 직원의 의사결정 강점/약점/보정 정확도를 정량화한 프로필. 지속적으로 업데이트.
- **알파/베타 분리**: 시장 노출(베타 — 결정 능력 아님)과 초과 수익(알파 — 결정 능력의 후보)을 통계적으로 분리.

Mauboussin의 연구: 투자에서 실력과 운을 통계적으로 분리하려면 **약 20년의 데이터**가 필요하다 (p < 0.05 수준). 개별 투자 결정의 평가는 근본적으로 불가능에 가깝다.

**군사 — After Action Review (AAR)**

- **승리 후에도 반드시 AAR을 실시한다** — 핵심 반(反)resulting 메커니즘.
- **관찰통제관(O/C)**: 외부의 훈련된 제3자가 "좋은 결과가 운 때문인지 적의 실수 때문인지" 판별.
- **결과가 아닌 결정 시점에 집중**: "그 순간에 어떤 정보를 갖고 있었나?"를 재구성.

한계: 전술적 결정(반복 가능)에는 잘 작동하지만, 일회성 전략적 결정에는 한계.

**의학 — M&M 컨퍼런스**

- 나쁜 결과를 세 가지로 분류:
  - 예방 가능한 부작용 (결정 오류)
  - 예방 불가능한 부작용 (불운 / 질병 진행)
  - 아차 사고 (결정 오류에도 좋은 결과 — "운 좋은 성공" 셀)
- **의사결정이 "옳은" 시점**: 결정 당시 이용 가능한 정보와 현재 근거 기반에 비추어 기대 효용을 최대화했다면, 결과와 무관하게 옳은 결정.

### 시간 지평선: 언제 평가하는가

| 분야 | 평가 시점 | 근거 |
|------|---------|------|
| 포커 | 수천 핸드 후 (개별 핸드 평가 무의미) | 통계적 수렴 필요 |
| VC | 7~12년 (펀드 수명 전체) | Kauffman Foundation: 대부분 VC 펀드가 공공 시장 인덱스도 못 이김 |
| 기업 전략 | 프로세스: 즉시 / 가정: 6~12개월 / 결과: 2~3년 | Lovallo & Sibony (McKinsey, HBR 2010) |
| 지정학 예측 | 1~5년 (1년 미만: 노이즈, 5년 초과: 인과 추적 불가) | Tetlock GJP |

### 실용적 결론: 개인 지식노동자가 쓸 수 있는 방법

연구의 엄밀성, 일회성 결정 적용 가능성, 실용성을 모두 통과하는 방법:

**방법 1. 프로세스 체크리스트 (가장 강력한 실증 근거)**

Kahneman/Lovallo/Sibony의 연구 — 1,000+개 기업 결정 분석 결과:
> **프로세스 품질이 결과에 미치는 영향이 분석 품질의 6배.**

결정 전에 확인:
- 진짜 다른 대안을 3개 이상 고려했는가?
- 반대 의견을 적극적으로 구했는가?
- 외부 관점(기저율)을 참조했는가?
- 핵심 가정 1~2개를 식별하고 무효화 조건을 정의했는가?
- 합리적인 정신 상태인가 (피로, 흥분, 매몰 비용 압박 없는가)?

이 체크리스트를 통과한 결정은, 결과와 무관하게, 통과하지 못한 결정보다 **대규모 표본에서** 더 좋은 결과를 낸다. **즉시 평가 가능.**

**방법 2. 가정 추적 (가장 빠른 피드백)**

결과를 기다리지 않고 **가정을 추적**한다:
1. 결정 시점: 핵심 가정 3~5개 기록
2. 각 가정에 대해: "이 가정이 틀리면 [구체적 관찰]이 나타날 것"
3. 30/90/180일 후 각 가정 점검

가정이 무너지면 결과 전에 업데이트할 수 있다. 군사 계획의 "분기 계획(branch planning)"과 동일 — 사전에 재평가 지점을 설정.

**방법 3. 보정 훈련 (유일하게 검증된 장기 개선법)**

Tetlock GJP 연구 — 보정 훈련이 예측 정확도를 ~15% 개선하며 효과가 지속됨.
100+개의 확률 예측을 축적해야 의미 있는 보정 데이터가 나오지만, 이건 **미래 의사결정 품질을 실제로 높이는 유일한 검증된 방법**이다.

### Overture 함의

> **가장 가치 있는 것: 의사결정 시점에 프로세스 품질을 강제하고, 다수 결정에 걸쳐 보정 데이터를 축적하는 것.**
>
> 결과 추적은 보정용으로 유용하지만, 개별 결정을 소급 재채점하는 데 사용해서는 안 된다.
> DQ scoring(프로세스 기반, 즉시)과 결과 추적(정보 제공, 지연)의 **명시적 분리**가 핵심 설계 원칙.

## 질문 2. 역사적 데이터셋은 존재하는가

### 솔직한 답: 범용 데이터셋은 없다. 도메인별 조각만 있다.

"전략적 결정 유형 → 결과 분포"를 매핑하는 통합 공개 데이터베이스는 **존재하지 않는다.**
아무도 만들지 못한 이유:

1. 결정을 맥락 간 일관되게 분류하기 어렵다
2. 결과가 수년 후에야 나타난다
3. 귀인 모호성 — 결과가 결정 때문인지 실행 때문인지 구분 불가
4. 조직이 실패 데이터를 자발적으로 공유하지 않는다
5. 분모 문제 — 극적이지 않은 결정은 기록되지 않는다

### 실제로 존재하는 것

| 데이터베이스 | 규모 | 공개? | 결정→결과 매핑? | 참조 클래스로 사용 가능? |
|------------|------|------|--------------|---------------------|
| **Flyvbjerg 메가프로젝트** | 16,000+ 프로젝트, 136개국, 1910~현재 | **비공개** (논문에서 요약 통계만) | 프로젝트 실행 결과 (비용/일정 초과) | 프로젝트 결정에 한해 가능 |
| **Good Judgment Project** | 수백만 예측, 500+개 질문/년 | **공개** (Harvard Dataverse) | 예측→해상도 | 보정용으로만 (결정 평가 아님) |
| **NTSB 사고 조사** | 1962~현재 전수 | **공개** (XML/CSV) | 실패 인과 체인 | 안전/실패 분석에 우수 |
| **FDA 약물 승인** | 1939~현재 | **공개** (Data.gov) | 승인 결정 + 시판 후 결과 | 제약 결정에 한해 가능 |
| **Standish CHAOS** | 50,000 IT 프로젝트 | **비공개** (요약만) | IT 프로젝트 결과 | **방법론 불투명, 신뢰도 논란** |
| **CB Insights 사후분석** | 483 스타트업 | 부분 공개 | 실패 원인 분류 | 생존자 편향 심각 |
| **PitchBook** | 156,000+ 펀드 | **비공개** (유료) | 투자→수익 | VC/PE 결정용 |
| **Paul Nutt 전략 결정** | 400+ 결정, 20년 추적 | **비공개** | **직접 결정→결과 매핑** | **가장 직접 관련되지만 접근 불가** |
| **Metaculus/Polymarket** | 수천 개 질문 | **공개** (API) | 예측→해상도 | 보정용 |
| **SPIVA (S&P)** | 전체 액티브 펀드 | **공개** | 운용→벤치마크 대비 성과 | 투자 결정용 |
| **군사 CALL 아카이브** | 미상 | 대부분 기밀 | 행동→결과 | 접근 극히 제한 |

### 실제로 확립된 기저율(Base Rates)

경험적으로 검증되어 있고, 발표된 기저율:

| 도메인 | 기저율 | 출처 |
|-------|-------|------|
| 메가프로젝트 비용 초과 | 91.5%가 예산/일정 초과 | Flyvbjerg, 16K 프로젝트 |
| 교통 프로젝트 비용 초과 | 중위 50~100% 초과 | Flyvbjerg |
| IT 프로젝트 성공 | 31% | Standish CHAOS |
| 대형 IT 프로젝트 성공 | <10% | Standish CHAOS |
| 전략적 결정 실패 | ~50% | Nutt, 400 결정 |
| M&A 가치 파괴 | ~61% 자본비용 회수 못함 | McKinsey |
| 볼트온 M&A 성공 | 80~85% | McKinsey |
| 액티브 펀드매니저가 벤치마크 상회 (15년) | ~10% | SPIVA/S&P |
| 스타트업 실패 (10년) | ~90% | 복수 출처 |
| 신약 Phase 1→승인 | ~10% | FDA |
| 레스토랑 폐업 (3년) | ~60% | BLS |

**이 기저율들은 실재하고 유용하지만, "결정 유형"이 아니라 "도메인"으로 분류되어 있다.**
"시장 진입 결정"이나 "채용 결정" 같은 범용 의사결정 유형에 대한 기저율은 존재하지 않는다.

### 분류 문제: 참조 클래스를 만들 수 있는가?

Flyvbjerg의 방식: **프로젝트 유형**(철도, 도로, 터널, IT)과 **지역**으로 분류.
이게 작동하는 이유: 결과 지표(비용 초과율)가 같은 유형 내에서 비교 가능하기 때문.

범용 전략적 결정에 적용하면:
- "시장 진입 결정" → 너무 넓다
- "동남아 시장 진입, 소프트웨어" → 데이터 포인트 ~20개
- "동남아 시장 진입, B2B SaaS, ARR $50M 미만" → 데이터 포인트 ~3개

**구체적일수록 유용하지만 데이터가 줄어든다. 이건 구조적 딜레마.**

### Cynefin / Thompson-Tuden 경험적 검증

Cynefin 프레임워크는 널리 사용되지만, **결과 데이터가 거의 없다.** "complex 도메인의 결정이 probe-sense-respond로 접근했을 때 성공률 X%"라는 데이터가 존재하지 않는다.

Thompson-Tuden은 300+개 전략적 결정에 대한 경험적 테스트가 하나 있다 — 상황 유형에 맞는 결정 접근법을 사용했을 때 결과가 유의미하게 개선됨. 하지만 단일 연구이며, 유지되는 데이터베이스가 아니다.

### 콜드 스타트 현실

**Day 1에 할 수 있는 것:**
- 위 표의 **발표된 기저율을 정적 참조 앵커로 임베딩**. M&A 결정 시 "61%가 자본비용 회수 못함"을 맥락으로 제시. 단순하고, 방어 가능하고, 즉시 유용.
- GJP 데이터를 **보정 방법론 참조**로 활용.
- 공개 API 데이터베이스(NTSB, FDA, Metaculus) 연결.

**Day 1에 할 수 없는 것:**
- "이런 유형의 결정은 성공률이 X%"라는 구체적 기저율 제공 — 그런 데이터가 없다.
- 자체 참조 클래스 구축 — 특정 카테고리에서 수백 건의 결과 추적 결정이 필요. 사용자가 연간 5~10건의 주요 결정을 내린다면, **개인 단위로는 수십 년** 소요.

**현실적 경로:**

```
Tier 1 (Day 1)     발표된 기저율을 정적 앵커로
                    "이런 결정에 대해 알려진 것"
                    ↓
Tier 2 (1~2년)     사용자 결정 일지 + 가정 추적 축적
                    아직 통계적 유효성 주장 안 함
                    "당신의 개인 결정 이력"으로 프레이밍
                    ↓
Tier 3 (3년+, 규모 확보 시)   수천 사용자의 결과 추적 데이터 집계
                              도메인별 독자 참조 클래스 구축 가능성
                              이것이 장기 해자
```

**참고:** Cloverpop($17M)이 기업 고객 대상 "Decision Bank"로 이 경로를 시도했으나, 통계적으로 유의미한 참조 클래스를 달성했다는 공개 증거는 없다.

## 질문 3. LLM 시뮬레이션으로 유의미한 판단 근거를 만들 수 있는가

### 먼저, 시뮬레이션이 작동하는 조건을 보자

역사적으로 시뮬레이션이 신뢰를 얻은 분야를 보면 공통 조건이 있다:

| 분야 | 신뢰도 | 왜 작동하는가 |
|------|-------|-------------|
| **기후 모델** | 높음 (50년 전 모델도 정확) | 잘 이해된 물리학 + 수백만 년 고기후 데이터로 검증 + **시스템이 예측에 반응하지 않음** |
| **금융 Monte Carlo** | 중간~높음 | 100+년 시장 데이터로 입력 분포 근거 + 잘 정의된 확률 과정 |
| **워게임 (군사)** | 예측은 낮음, 준비도는 높음 | RAND: "정량적 결과가 아닌 질적 통찰 생성" + WWII 해군 장교 거의 전원 참여 |
| **역학 모델 (COVID)** | 1~2주: 합리적 / 6주+: 불량 | CDC 모델 25%만 naive 기준선 상회 + 정책 변화(반사성)로 예측 붕괴 |
| **Agent-Based 경제 모델** | 정성적 통찰은 유용, 정량 예측은 약함 | Santa Fe SFI-ASM: "정량적 값이 실제 데이터와 잘 맞지 않음" + 파라미터 민감도가 "아킬레스건" |

**공통 패턴:**
- 작동 조건: 알려진 물리학 + 경험적 입력 분포 + 비반사적 시스템 + 제한된 파라미터
- 실패 조건: 반사적 시스템(인간 행동이 예측에 반응) + 신규 상황 + 높은 파라미터 민감도

**의사결정 환경은 거의 모든 실패 조건을 충족한다.** 반사적이고, 종종 신규 상황이며, 파라미터가 많다.

### LLM 시뮬레이션의 구체적 연구 결과

**Generative Agents (Stanford "Smallville", Park et al. 2023):**
- 크라우드워커가 에이전트 인터뷰 응답을 인간보다 더 believable하다고 평가
- 하지만 물리적/맥락적 제약을 잘못 분류 (점유된 욕실 진입, 닫힌 가게 방문)
- **사회적 스크립트 모델링은 우수, 실제 제약 모델링은 취약**

**LLM-as-Judge 연구 (2024 종합 서베이):**
- LLM이 **자기 출력을 인식하고 선호**한다 (self-recognition↔self-preference 선형 상관)
- 장황하고 형식적인 출력을 내용 품질과 무관하게 선호
- 참조 답변과 점수 설명이 없으면 인간 판단과의 정렬 급격히 저하
- **시나리오를 생성하고 같은 LLM이 평가하면 폐쇄 루프 편향 문제 발생**

**Multi-Agent Debate (ICLR 2025 분석):**
- **"대부분의 MAD 프레임워크가 Chain-of-Thought보다 일관되게 더 좋은 성과를 내지 못한다"**
- 검증 가능한 정답이 있는 과제(수학, 사실 질문)에서는 효과
- **개방형 판단 과제에서는 일관된 이점 없음**

**LLM 보정 연구 (2025):**
- LLM이 84.3%의 경우에서 과신(overconfident)
- Dunning-Kruger 패턴: **가장 틀릴 가능성이 높을 때 가장 높은 확신을 표현**

**LLM 반사실적 추론 (EMNLP 2024):**
- 구문적으로 올바른 반사실 생성은 잘함
- **의미적, 맥락적 일관성은 복잡한 시나리오에서 실패**
- "사소하게 타당한" 반사실에 편향 — 진짜 도전이 아니라 표면적 변형

**LLM 예측의 방향성 vs 크기 (Horton, NBER 2023):**
- LLM이 효과의 **방향(부호)**은 신뢰성 있게 예측
- 효과의 **크기(magnitude)**는 신뢰성 있게 예측하지 못함
- **이론적 구조 파라미터를 조건으로 제공하면 예측이 극적으로 개선**

### "73%의 시뮬레이션 시나리오에서 주요 리스크는 X였다" — 이게 유용한가?

**세 가지 답:**

**(a) 이 형태 그대로 유용하고 방어 가능한가? — 아니다.**

"73%"는 보정된 확률을 암시한다. LLM 시뮬레이션은 보정된 확률을 제공할 수 없다.
LLM 생성 시나리오의 빈도 분포를 유효한 확률 분포로 취급할 근거가 없다.
시나리오는 **실제 결과 분포가 아니라 인터넷 텍스트의 분포**를 반영한다.
성공한 시장 진입 사례가 실패 사례보다 출판물에 압도적으로 많다 — 생존자 편향이 LLM에 내장된다.

**(b) 위험하게 오해의 소지가 있는가? — 숫자 확률로 제시하면 그렇다.**

- 사용자가 "73%"에 앵커링하여 보험통계학적 수치처럼 취급할 것
- 이 숫자는 경험적 보정이 없다 — LLM 생성 텍스트에서 특정 패턴의 빈도일 뿐
- McNamara의 오류: 정량화할 수 없는 것을 정량화한 후 그 숫자를 진리로 취급
- **아예 숫자가 없는 것보다 더 위험할 수 있다** — 거짓 확신을 부여하므로

**(c) 적절히 프레이밍하면 유용한가? — 예. 이것이 올바른 답이다.**

```
잘못된 프레이밍:
"시뮬레이션 시나리오의 73%에서 주요 리스크는 X였다"

올바른 프레이밍:
"다양한 가정 변형 하에서 당신의 계획을 스트레스 테스트한 결과,
가장 빈번하게 부상한 리스크는 X였다.
이 리스크는 다양한 이해관계자 관점과 가정 변형에 걸쳐 반복적으로 나타났다.
다음 조건에서 악화된다: [...]"
```

차이:
- 가짜 확률 백분율 없음
- "구조화된 탐색"으로 명시적 프레이밍, "예측"이 아님
- **어떤 리스크**에 초점, **얼마나 가능성 있는지**가 아님
- 실행 가능 ("악화 조건" 제시)

### 시뮬레이션이 실제로 할 수 있는 것과 없는 것

**증거가 지지하는 것:**

| 할 수 있는 것 | 근거 |
|-------------|------|
| 개인이 혼자 고려할 것보다 **더 넓은 리스크/실패 모드 공간** 표면화 | Klein: 사전부검으로 리스크 식별 +30% |
| 결정자의 정신 모델을 확장하는 **질적으로 그럴듯한 이해관계자 반응** 생성 | Park et al. 2023: believable 행동 시뮬레이션 |
| 가정 변형에 걸쳐 어떤 리스크가 지배적인지 **방향적 지표** 제공 | Horton 2023: 방향은 맞추지만 크기는 못 맞춤 |
| 다양한 "페르소나"를 **집계하면 개선** | Schoenegger et al. 2024 (Science Advances): 12개 LLM 앙상블이 925명 인간 군중과 통계적으로 구별 불가 |
| 가정을 변화시키며 결론이 바뀌는지 **견고성 스트레스 테스트** | Shell 시나리오 플래닝 전통 |

**증거가 반대하는 것:**

| 할 수 없는 것 | 근거 |
|-------------|------|
| 실제 결과에 대한 **보정된 확률** 산출 | LLM 84.3% 과신 (2025 보정 연구) |
| 경험적 참조 클래스 데이터 **대체** | Flyvbjerg 기저율은 16K+ 실제 프로젝트에서 도출 |
| 훈련 데이터에 전례 없는 **진짜 신규 상황** 시뮬레이션 | Agent-based 모델 한계 동일 |
| 실제 인간 응답의 **분산 재현** | Argyle et al. 2023: 분포가 실제보다 좁고 정형화됨 |
| **진정한 반대 의견** 생성 | Nemeth 2001: 배정된 역할은 진짜 반대보다 약함; LLM은 구조적으로 "배정된" 비평가 |

### 신뢰할 수 있는 대안: 예측이 아닌 준비

연구가 수렴하는 방향은 **예측(prediction)**이 아닌 **준비(preparedness)**다.

**Shell 시나리오 플래닝의 교훈:**
Shell은 1973년 석유 위기를 *예측*하지 않았다. 석유 가격 급등 시나리오에 대한 **대응 계획을 사전에 수립**해 놓았기 때문에 위기 징후가 나타나자 가장 먼저 움직였다.

Pierre Wack의 핵심 통찰:
> "시나리오는 의사결정자의 **정신 모델을 바꿈**으로써 작동한다. 예측을 제공하는 것이 아니다."

이건 LLM 시뮬레이션이 **실제로 할 수 있는 것**과 정확히 일치한다.

**실용적 설계 원칙:**

1. **"구조화된 탐색"으로 프레이밍, 절대 "예측"이 아님**
2. **시뮬레이션에서 숫자 확률을 출력하지 않음** — "지배적 리스크", "부상하는 패턴", "가정 민감도" 같은 질적 지표 사용
3. **가능한 곳에서 실제 데이터로 근거화** — Horton: "구조적 파라미터를 조건으로 제공하면 극적으로 개선"
4. **다양한 모델 설정을 집계** — Schoenegger et al.: 앙상블이 인간 군중과 맞먹음; 단일 모델은 못 맞춤
5. **프레이밍을 사용자에게 명시** — "이것은 예측이 아닙니다. 당신이 놓치고 있을 수 있는 것을 구조적으로 찾는 방법입니다."
6. **절대 판단이 아닌 상대 비교 사용** — "Plan A가 Plan B보다 더 많은 스트레스 시나리오에서 살아남았다"는 방어 가능. "Plan A의 성공률 73%"는 방어 불가.

## 종합: 세 질문에 대한 솔직한 답

### Q1. 옳고 그름의 기준

**프로세스를 평가하라, 결과가 아니라.** 이것은 포커, 투자, 의학, 군사 모든 분야가 수렴하는 답이다. 결과로 개별 결정을 평가하는 것은 인지 오류이며, 통계적으로도 불가능하다(투자에서 20년 필요). Overture의 DQ scoring이 이미 이 방향에 있지만, **결과를 DQ 재채점에 사용하는 게 아니라 보정 데이터로 사용**해야 한다.

### Q2. 역사적 데이터셋

**범용 데이터셋은 없다. 도메인별 기저율은 있다.** Day 1 전략은 발표된 기저율을 정적 앵커로 제공하는 것이다. 자체 데이터 해자는 Tier 2~3에서 축적하되, **개인 단위로는 수십 년이 걸리고 집계 단위에서도 수천 사용자가 필요하다.** 이건 장기 전략이지 단기 경쟁 우위가 될 수 없다.

### Q3. LLM 시뮬레이션

**예측으로는 불가. 구조화된 탐색으로는 가치 있다.**
- 숫자 확률 → 위험 (McNamara 오류 + LLM 84.3% 과신)
- "어떤 조건에서 이 계획이 무너지는가?" → 유용 (Shell 시나리오 플래닝 전통)
- 다양한 LLM 페르소나 앙상블 → 925명 인간 군중 수준 (Schoenegger et al.)
- 실제 데이터로 근거화하면 극적 개선 (Horton NBER 2023)

### Overture에 대한 전체 함의

Overture의 가치는 **"미래를 예측해 주는 도구"가 아니라 "당신이 구조적으로 놓칠 수밖에 없는 것을 찾아주는 도구"**에 있다.

이건 약화가 아니라 강화다:
- 예측 도구는 틀리면 신뢰를 잃는다
- 탐색 도구는 한 번이라도 사각지대를 찾으면 가치가 입증된다
- WHO 체크리스트는 사망률을 47% 줄였다 — 예측이 아니라 **체계적 비망각**의 가치

```
Overture의 포지셔닝:

[예측 도구]                                              [탐색/보정 도구]
"73% 확률로 성공"                                    "이 세 가지를 놓치고 있다"
Monte Carlo / AutoML                                 Pre-mortem / Scenario Planning
Karpathy Loop                                        Overture 4R
보정된 입력 분포 필요                                    프로세스 품질만으로 가치 창출
틀리면 신뢰 상실                                       한 건만 찾아도 가치 입증

                                                      ← Overture는 여기
```
