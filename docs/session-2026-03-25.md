## 전략 논의 (구현 전)

### 1. 배포 형태 (MCP/Plugin vs 웹앱)
- MCP/Plugin으로 배포하면: 프롬프트 노출 + 무료 사용 + 데이터 수집 실패가 동시 발생
- sync-back 모델도 문제: "왜 내 데이터를 저기로 보내야 하지?" 거부감
- **결론: 웹앱이 본진. MCP는 나중에 맛보기용으로만.**

### 2. LLM 포장지 탈출
- 현재 Overture는 구조적으로 LLM 의존도가 높음 (80%)
- 살아남는 AI 제품 패턴: LLM이 전체 가치의 작은 부분 (Cursor, Perplexity 등)
- **해자는 기술이 아니라 "이 방법이 효과가 있다"는 증거 + 누적 데이터**

### 3. Anthropic 매력도 분석
- 시장에 "구조화된 다관점 AI 의사결정 제품" = 0개
- "AI 악마의 대변인 제품" = 0개
- Anthropic이 못 풀고 있는 문제(sycophancy)를 제품 레이어에서 우회 → 보완적 가치
- 가장 현실적 경로: 리서치 가치 (B경로) — "Overture가 효과가 있다"는 데이터

### 4. 메타 레이어 포지셔닝
Anthropic "Long-running Claude" 논문 리뷰 후 도출:
> "AI가 더 자율적으로 일할수록, 시작점의 판단이 더 중요해진다. Overture는 그 시작점을 설계하는 메타 레이어다."

- 에이전트 실행 전 CLAUDE.md/성공기준/체크포인트 설계 = Overture의 4단계
- 인간 개입 "최소화"가 아니라 인간 판단의 "밀도"를 높이는 도구
- 4단계는 줄이지 않되, 데이터 축적으로 속도와 마찰을 줄이는 방향

### 5. 정체성
- 내부: "판단 설계 메타 레이어"
- 외부: "생각을 뾰족하게" (Threads 후기에서 나온 실제 사용자 언어)
- Hero 카피: "AI에게 시키기 전에, 생각을 뾰족하게."

## 리서치 (40편 논문)

### 3개 영역으로 병렬 검색
1. LLM 아첨(Sycophancy) — Anthropic 자체 논문 + 구조적 완화
2. 구조화된 추론/의사결정 — CoT, 멀티에이전트, 구조화 vs 자유대화
3. 페르소나 시뮬레이션 + 인지편향 — LLM 역할극, 프리모템, DQ 측정

### 핵심 발견
- Dell'Acqua/Harvard-BCG (758명): 구조화 AI가 자유대화 대비 12% 우수
- Du/MIT: 멀티에이전트 토론 → 추론 정확도 8%p 향상
- Sharma/Anthropic: 58% 상황에서 아첨, 78.5%는 구조적 문제
- Klein/HBR: 프리모템 → 위험 식별 30% 향상
- Spetzler: DQ 6요소 — 결과와 독립적 프로세스 품질 측정

### Overture의 고유 연구 질문
> "개별 기법(CoT, 멀티에이전트, 프리모템, 악마의 대변인)을 하나의 워크플로우로 통합했을 때 시너지가 있는가?"

## 구현

### Phase 0: 전체 단계 Eval 확장
- `eval-engine.ts`: Recast(4) + Rehearsal(4) + Refinement(4) = 12개 신규 eval (총 16개)
- `prompt-mutation.ts`: 16개 eval 전체 자동 보정 지시

### Phase 1: Outcome Tracking
- `outcome-tracker.ts` (신규): 실행 결과 기록, 페르소나 예측 정확도, 가설 검증, 성공률
- `types.ts`: OutcomeRecord, MaterializedRisk, ApprovalOutcome

### Phase 2: 학습 루프 완성
- `retrospective.ts`: 답변 저장 + 액셔너블 인사이트 추출
- `context-builder.ts`: Outcome 인사이트 + Retrospective 답변 → AI 프롬프트 자동 주입
- `reframing-strategy.ts`: eval 성과 데이터 기반 전략 선택 (규칙 기반 fallback)

### Phase 3: Decision Quality 측정
- `decision-quality.ts` (신규): DQ 6요소 자동 계산 + 트렌드 분석 + DQ↔Outcome 상관

### UI 연결
- `RecastStep.tsx`: 완료 시 recast eval 기록
- `PersonaFeedbackStep.tsx`: 피드백 완료 시 rehearsal eval 기록
- `RefinementLoopStep.tsx`: 수렴 시 refinement eval + DQ 스코어 + 회고 답변 저장 + Outcome Recording UI + DQ Score Card

### 보상 체계
| 단계 | 보상 |
|------|------|
| 악보 해석 | before/after 질문 변화 시각화 + "전제 N건 중 M건 불확실 → 다음 단계에서 검증" |
| 편곡 | "N곳에서 당신의 판단이 필요합니다. 나머지는 AI가 실행합니다" |

| 합주 | DQ 스코어 바 차트 (6요소) + anti-sycophancy 칩 (사각지대, 프레이밍 도전, 판단 수정) |
