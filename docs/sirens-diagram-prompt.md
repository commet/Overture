# 모식도 제작 프롬프트 (claude.ai용)

## 요청

LinkedIn 게시글에 첨부할 모식도를 그려줘. 오디세우스/세이렌 신화와 멀티에이전트 아키텍처의 context 전략을 대응시키는 다이어그램이야.

## 전체 구조

가로로 긴 직사각형 이미지 (LinkedIn 최적 비율 1200x627 또는 유사한 가로형).
배경은 짙은 남색(#0f172a 정도, 바다를 암시).

### 상단 영역

화면 상단 중앙에 "세이렌의 노래 (AI Capability)" 텍스트.
그 아래에 물결 패턴이나 음파 패턴을 가로로 넓게 깔아서 "노래가 퍼지고 있다"는 느낌.

### 중단 영역 — 핵심 (배 4척)

4개의 배를 가로로 나란히 배치. 각 배는 단순화된 범선 실루엣이나 상단에서 내려다본 배 모양(타원형).

각 배 안에 선원 4명을 아이콘으로 표시 (원형 아이콘 4개).
선원 아이콘의 색상/상태로 context 전략을 표현:
- 빨간색 또는 X 표시 = 차단됨 (밀랍, narrow context)
- 초록색 또는 열린 귀 = 열림 (full context)  
- 노란색 또는 문서 아이콘 = 구조화된 기록 (structured artifact)

배 1: CrewAI / Anthropic — "전원 밀랍"
- 선원 4명 전부 빨간색(차단)
- 배 아래에 레이블: "CrewAI / Anthropic"
- 그 아래 작은 텍스트: "전원 밀랍 — 노만 젓는다"

배 2: AutoGen / OpenAI — "밀랍 없음"
- 선원 4명 전부 초록색(열림)
- 레이블: "AutoGen / OpenAI"
- 작은 텍스트: "밀랍 없음 — 모두 듣는다"

배 3: MetaGPT — "항해 기록"
- 선원 4명 전부 노란색(문서)
- 레이블: "MetaGPT"
- 작은 텍스트: "항해 기록 — 기록을 읽는다"

배 4: Overture(또는 "Adaptive") — "역할별 적응"
- 선원 4명이 각각 다른 색상:
  - 정찰병(Research): 빨간색 — "minimal"
  - 항해사(Data): 노란색 — "structured" 
  - 망보기(Critic): 초록색(하지만 동료 결과만) — "focused"
  - 기록관(Writer): 초록색 — "full"
- 레이블: "Adaptive (Overture)"
- 작은 텍스트: "역할별 적응 — 각자 다르게 듣는다"

이 네 번째 배만 약간 강조 — 테두리를 밝게 하거나 살짝 크게.

### 각 선원 아이콘 옆에 역할 레이블 (네 번째 배에만)

네 번째 배에만 각 선원 옆에 작은 레이블:
- 🔍 정찰병 (Research) — minimal
- 📊 항해사 (Data) — structured
- 👁 망보기 (Critic) — focused
- ✍️ 기록관 (Writer) — full

### 하단 영역

화면 하단 중앙에 작은 섬 아이콘과 "이타카 (Decision)" 텍스트.
4척의 배에서 이타카를 향해 아래로 향하는 화살표나 경로선.

### 범례 (우측 하단 또는 좌측 하단에 작게)

- 🔴 차단 (narrow context)
- 🟢 개방 (full context)  
- 🟡 구조화 (structured artifact)

## 스타일 가이드

- 전체 톤: 깔끔하고 모던한 다이어그램. 인포그래픽 느낌.
- 폰트: 산세리프. 한글과 영어 병기.
- 색상: 어두운 남색 배경에 밝은 요소들. 미니멀.
- 배 디자인: 너무 사실적이지 않게 — 기하학적으로 단순화된 범선 실루엣이나 위에서 내려다본 타원형.
- 선원 아이콘: 원형 안에 사람 실루엣, 또는 단순한 원형 도트로도 충분.
- 전체적으로 "테크 블로그의 아키텍처 다이어그램" 느낌이되, 신화적 요소(배, 물결, 섬)가 은은하게 녹아있는 톤.

## 핵심 전달 메시지

이 다이어그램을 3초 보고 알 수 있어야 하는 것:
1. 멀티에이전트 시스템마다 context 전략이 다르다
2. 대부분은 모든 에이전트에게 같은 전략을 쓴다
3. 네 번째(Adaptive)만 역할별로 다른 전략을 쓴다
4. 모든 배의 목적지는 이타카(Decision)다
