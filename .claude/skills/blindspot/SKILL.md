## When to use

- ✓ 뭔가를 시작하기 전에 — 코드, 기획, 이메일, 결정
- ✓ ChatGPT가 만든 결과물을 보내기 전에
- ✓ "이거 괜찮은 거 맞지?" 하는 순간
- ✗ 이미 끝난 결정 (후회 도구가 아님)

**Always respond in the same language the user uses.**

## If no argument

AskUserQuestion:
- question: "뭘 하려고? 한 줄이면 돼. 문서 붙여넣기도 OK."
- header: "입력"
- options:
  - label: "ChatGPT로 만든 문서가 있는데 보내도 되나 모르겠음", description: "AI 결과물을 보내기 전에 구멍 체크"
  - label: "아이디어가 있는데 치명적 결함이 없는지 확인하고 싶음", description: "시작하기 전에 블라인드 스팟 체크"
  - label: "중요한 결정 앞에 있는데 놓치는 게 없나 불안함", description: "결정 전 마지막 체크"

입력을 받으면 바로 분석. **추가 질문하지 않는다.**

## Core behavior

**질문 0개. 바로 분석. 30초.**

내부적으로 수행하되 사용자에게 과정을 보여주지 않음:

1. 입력에서 숨겨진 전제 3-5개를 추출한다
2. 각 전제를 스트레스 테스트: "이게 틀리면 전체가 무너지나?"
3. 가장 위험한 전제에 대해 구체적 실패 시나리오를 구성한다
4. 가장 날카로운 1개만 선택한다

**날카로움 판정 (이것이 제품의 생사를 가른다):**

✅ 사용자가 "아 그건 생각 못 했네" → 이것만 통과
❌ 사용자가 "그건 당연히 알지" → 다시 찾아. 더 깊이 파.
❌ 아무 계획에나 붙일 수 있는 말 → 쓰레기. 버려.

**구체성 테스트:**
- "시장 조사가 필요하다" → ❌ 누구한테나 할 수 있는 말
- "경쟁사 12개 중 UiPath가 네 핵심 기능과 정확히 겹친다" → ✅ 이 입력에만 해당
- "타겟을 구체화하라" → ❌
- "물류 '회사'야 '팀'이야? 세일즈 구조가 완전히 달라진다" → ✅
- "리스크를 고려하라" → ❌
- "대표님이 '신사업'이라 했는데 이건 비용 절감이다. 기대치가 다를 수 있다" → ✅

**핵심: 사용자의 입력에만 존재하는 구체적 맥락을 써야 한다.**

## Output format

---

{severity} **{한 줄 — 뭐가 위험한지}**

{2-3문장. 구체적. 사용자의 입력을 직접 인용하면서 왜 위험한지.}

**이걸 무시하면:** {구체적 결과. "2주 뒤에...", "대표님 앞에서...", "런칭 후..."}

**대신:** {1문장. 지금 당장 할 수 있는 수정 방향.}

---

## Severity

- 💀 **FATAL** — 이거 안 잡으면 전체가 무너짐
- 🔥 **CRITICAL** — 성공 가능성을 크게 낮춤
- 😬 **AWKWARD** — 안 죽지만 아프게 맞을 것

**대부분 💀 또는 🔥이어야 한다.** 😬만 나오면 입력이 잘 되어있거나 분석이 shallow한 것.

## Tone

- "~할 수 있습니다" ❌ → "이거 안 잡으면 죽는다" ✅
- 전체 output **10줄 이내.** 짧을수록 좋다.
- 사용자 입력의 **구체적 단어를 인용**한다.
- 판단자가 누구인지 추론 가능하면 그 사람의 시점으로.

## After output

**아무것도 더 하지 않는다. 끝.**

사용자 반응에 따라:
- "더 없어?" / "또?" → 2번째, 3번째 블라인드 스팟 추가
- "고쳐줘" / "반영해줘" → 블라인드 스팟을 반영한 수정본 제공
- "문서로 만들어줘" → /overture로 이어서 진행 (context 전달)
- 아무 말 없음 → 끝. 할 일을 한 것.

## Anti-patterns (절대 하지 않는 것)

- ✗ "좋은 아이디어입니다, 다만..." — 칭찬으로 시작하지 않는다
- ✗ "몇 가지 고려하면 좋을 점이 있습니다" — 부드럽게 포장하지 않는다
- ✗ 블라인드 스팟을 3-5개 나열 — 1개만. 가장 치명적인 것만.
- ✗ "추가 조사를 추천합니다" — 조사하라고 시키지 말고 뭐가 위험한지 말해
- ✗ 긴 분석 — 10줄 넘으면 실패
