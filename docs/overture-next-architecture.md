## 한 문단 요약

Overture의 가치는 "미래를 예측해 주는 도구"가 아니라 **"당신이 구조적으로 놓칠 수밖에 없는 것을 찾아주는 도구"**에 있다. 이를 위해 세 가지가 상호 강화적으로 작동해야 한다: (A) 공개 연구 기반의 구조적 참조 시스템이 가정 트리의 초기 조건을 풍부하게 하고, (B) 진단적 증거와 반복 대화 기반의 고도화된 비평이 신호를 생성하며, (C) Backward-Forward Induction Loop이 A와 B를 엮어 수렴하는 판단 프로세스를 만든다.

## 1. 전체 구조: 왜 A, B, C는 하나의 시스템인가

```
┌─────────────────────────────────────────────────────────┐
│                  C. Backward-Forward Loop                │
│                  (전체를 엮는 프레임워크)                  │
│                                                          │
│   ┌─── BACKWARD 단계 ───┐    ┌─── FORWARD 단계 ───┐     │
│   │                      │    │                     │     │
│   │  "성공하려면 뭐가    │    │  "관찰에 비추어     │     │
│   │   참이어야 하는가?"  │    │   가정을 업데이트"  │     │
│   │                      │    │                     │     │
│   │   ▲ A가 여기를       │    │   ▲ B가 여기를      │     │
│   │     풍부하게 한다    │    │     고도화한다      │     │
│   └──────────────────────┘    └─────────────────────┘     │
│              │                         │                  │
│              └────── 반복/수렴 ─────────┘                  │
│                                                          │
│   Reframe ──→ Recast ──→ Rehearse ──→ Refine             │
│   (backward)  (multi)   (forward)   (iterate)            │
└─────────────────────────────────────────────────────────┘
```

**상호 강화 관계:**

| | A를 강화 | B를 강화 | C를 강화 |
|---|---------|---------|---------|
| **A (구조적 참조)** | — | 페르소나에게 근거 데이터 제공 | 가정 트리의 사전 확률을 실증 기반으로 |
| **B (고도화된 비평)** | 사용자 반응 데이터가 A에 축적 | — | Forward 단계의 신호 품질 향상 |
| **C (BF Loop)** | 어떤 참조 데이터가 필요한지 식별 | 비평의 구조와 타이밍을 규정 | — |

**이것이 분리된 세 프로젝트가 아니라 하나의 시스템인 이유:**
- A 없이 C를 돌리면 → 가정의 사전 확률이 순전히 LLM 추측 (84.3% 과신)
- B 없이 C를 돌리면 → Forward 단계의 신호가 표면적 역할극 수준
- C 없이 A+B만 있으면 → 좋은 정보와 비평이 있지만 수렴하는 구조가 없음

## 2. A: 구조적 참조 시스템

### 문제
범용 의사결정 데이터셋은 존재하지 않는다. "전략적 결정 유형 → 결과 분포"를 매핑하는 공개 데이터베이스는 아무도 만들지 못했다.

### 해법: 도메인이 아니라 구조로 분류한다

인지과학이 보여주는 것: 전문가는 **구조적 유사성**으로 문제를 분류하고, 초보자는 **표면적 유사성**(도메인)으로 분류한다. 구조적 분류가 전이(transfer) 성공을 예측한다 (Holyoak & Koh 1987, Chi et al. 1981).

→ "M&A 결정"이 아니라 "불가역적 결정 / 시간 압박 / 단일 옵션 고려 / edict 실행"으로 분류하면, 공개 연구 데이터만으로 의미 있는 참조를 제공할 수 있다.

### 5-Layer 아키텍처

**Layer 1: 도메인별 기저율 (정적 앵커)**

의사결정 유형을 감지하면 자동으로 관련 기저율을 표면화.

| 도메인 | 기저율 | 출처 | 신뢰도 |
|-------|-------|------|-------|
| M&A (변혁적) | ~70% 가치 파괴 | McKinsey | 높음 |
| M&A (볼트온, 같은 산업) | 80~85% 성공 | McKinsey | 높음 |
| 메가프로젝트 예산 내 완료 | 8.5% | Flyvbjerg, 16K 프로젝트 | 높음 |
| IT 프로젝트 완전 성공 | 31% | Standish CHAOS | 중간 (방법론 논란) |
| 대형 IT 프로젝트 성공 | <10% | Standish CHAOS | 중간 |
| 전략적 결정 (일반) | ~50% 실패 | Nutt, 376 결정 | 높음 |
| 액티브 펀드매니저 > 벤치마크 (15년) | ~10% | SPIVA/S&P | 높음 |
| 스타트업 생존 (10년) | ~10% | 복수 출처 | 높음 |
| 신약 Phase 1 → 승인 | ~10% | FDA | 높음 |

사용 방식: "당신의 결정은 [M&A / 프로젝트 / ...] 영역에 해당합니다. 이 영역의 역사적 기저율: [X%]. 이것은 예측이 아니라 외부 관점 앵커입니다."

**Layer 2: 프로세스-구조 리스크 플래그**

Nutt의 376개 전략 결정 + Lovallo-Sibony의 1,048개 결정에서 도출.

| 구조적 특징 | 경험적 패턴 | 감지 시 행동 |
|-----------|-----------|-----------|
| Edict 전술 (그냥 시행 선언) | 38% 성공 | "Intervention(90%)이나 Participation(80%) 고려를 권합니다" |
| 단일 옵션만 고려 | 성공률 극적 하락 | "최소 3개 대안 탐색을 강제합니다" |
| 맥락-접근법 미스매치 | 60%가 미스매치, 성공 유의미 하락 | Thompson-Tuden 정합성 확인 |
| 프로세스 품질 낮음 | 분석 품질의 6배 영향 | Kahneman-Sibony 체크리스트 실행 |

감지 방식: Reframe 단계에서 사용자의 의사결정 구조를 분석할 때, 위 특징을 자동으로 스캔.

**Layer 3: 편향 감지 (구조 조건부)**

모든 편향을 나열하는 게 아니라, **이 결정의 구조적 특징에서 가장 활성화될 가능성이 높은 편향**만 표면화.

| 구조적 조건 | 활성화되는 편향 | 정량적 근거 |
|-----------|-------------|-----------|
| 비모듈적 + 신규 프로젝트 | 계획 오류 (fat-tail) | IT: 평균 73% 초과, 상위 18%는 447% (Flyvbjerg) |
| 90% 확신 추정 | 과신 | 실제 40~60% 정확 (Russo & Schoemaker, 1,000+ 관리자) |
| 대규모 금전 투자 + 최근 지출 | 매몰 비용 | 금액↑ + 시간↓ = 효과 최대 (메타분석, Roth et al.) |
| 대안이 많음 + 선호 약함 | 현상유지 편향 | 대안 수와 비례 강화 (Samuelson & Zeckhauser) |
| 느린 피드백 환경 | 전문가 과신 | 의사/주식중개인: 확신↑ 정확도→ (Kahneman & Klein 2009) |

감지 방식: Backward 단계에서 가정 트리 생성 시, 각 가정의 구조적 특징을 분석하여 관련 편향을 태그.

**Layer 4: 맥락-접근법 정합성 체크**

Thompson-Tuden 매트릭스의 유일한 실증 검증 (Nutt 2002, 376 결정):

```
                    수단에 대한 합의
                    알려짐        불확실
목표에 대한   명확   계산적         판단적
합의                (최적화)       (전문가 판단)
              불명확  타협적         영감적
                    (협상)         (리더십 비전)
```

실제로 60%가 미스매치. 매칭 시 성공률 유의미 향상. → Reframe에서 자동 분류 후 접근법 권고.

**Layer 5: 개인 보정 (축적형)**

Tetlock GJP: 100+개 확률 예측 축적 시 보정 정확도 +15%, 효과 지속.
→ 사용자가 각 가정에 확률을 부여할 때마다 기록. 시간이 지나면 "당신은 90% 확신일 때 실제로 55% 정확합니다"를 보여줄 수 있다.

| 우선순위 | 레이어 | 난이도 | Day 1 가능? | 가치 |
|---------|-------|-------|-----------|------|

## 3. B: 고도화된 비평 시스템

### 문제

LLM 페르소나는 구조적으로 "배정된 악마의 대변인"이다. Nemeth(2001): 배정된 반대는 인지적 강화(bolstering)를 유발 — 사용자가 오히려 자기 입장을 굳힌다.

### 해법: 다섯 가지 메커니즘으로 간극을 좁힌다

**메커니즘 1: 진단적 증거 (Diagnostic Evidence)**

Heuer의 ACH 원리를 적용. 비평이 **가설들을 구별**할 수 있을 때만 가치가 있다.

```
비진단적 (현재):
  "실행 리스크가 있습니다"
  → 모든 계획에 해당. 사용자 반응: "알고 있어요"

진단적 (목표):
  "당신의 계획은 데이터팀이 2주 내 통합을 완료한다고 가정합니다.
   이 가정이 틀리면 전체 일정이 6주 지연됩니다.
   근거: 현재 스프린트 백로그 분석 결과 6주 분량 적재."
  → 특정 가정 공격. 사용자 반응: "그건 확인해봐야겠네"
```

구현: 페르소나 출력 스키마에서 `untested_assumptions`를 1순위 필드로 승격. 각 비평에 **"공격하는 가정"**과 **"가정이 틀렸을 때의 구체적 결과"**를 필수로 요구.

증거 수준: Heuer CIA SATs, RAND "더 넓은 범위의 결과를 다룸"

**메커니즘 2: Chain-of-Verification (CoVe)**

Meta/FAIR (ACL 2024). 페르소나 비평 생성 후 **독립적 검증 단계** 추가.

```
Step 1: 페르소나가 비평 생성
  "이 타임라인은 비현실적입니다"

Step 2: 검증 (비평을 보지 않고 독립적으로)
  "계획의 타임라인 관련 구체적 근거를 찾아라.
   어떤 가정이 타임라인을 지탱하는가?
   그 가정에 대한 증거와 반증은?"

Step 3: 검증 결과로 비평 수정 또는 폐기
```

핵심: Step 2에서 원래 비평을 참조하면 같은 환각을 복사. **독립 실행**이 필수.

증거 수준: CoVe가 zero-shot, few-shot, CoT 모두 상회 (ACL 2024)

**메커니즘 3: 반복적 대화 (Iterative Dialectic)**

단일 패스 비평 → 사용자 반박 → 페르소나 재응답의 **진짜 루프**.

```
라운드 1: 페르소나 비평 → 사용자 반박
라운드 2: 페르소나가 반박을 수용하거나 더 강한 근거로 재반박
라운드 3: 수렴 또는 "해소되지 않는 핵심 긴장" 식별
```

근거:
- Schwenk/Schweiger: 변증법적 탐구 > 악마의 대변인 (가정 표면화에서)
- Scalable Delphi (2025): 다중 라운드 반복 후 LLM 패널이 인간 전문가 패널과 Pearson 상관 0.95
- 경험이 쌓일수록 효과 향상 (Schweiger: 연습한 그룹의 결정 품질 상승)

현재 간극: Refine이 부분적으로 하지만, **사용자의 실시간 반박에 페르소나가 응답하는 대화**가 없다. 이것이 가장 큰 품질 향상 지점.

증거 수준: Scalable Delphi Pearson 0.95, Schwenk 실험적 검증

**메커니즘 4: 이질적 모델 앙상블**

같은 LLM에 다른 프롬프트보다, **다른 LLM**을 페르소나로 사용.

- 동질 앙상블: 82% 정확도 (GSM-8K)
- 이질 앙상블: **91%** (Gemini + PaLM + Mixtral)
- 12개 LLM 앙상블 ≈ 925명 인간 군중 (Schoenegger et al., Science Advances 2024)

구현: 기술적으로는 다중 API 호출. 비용 증가가 있지만, 핵심 결정에 대해서는 가치 있을 수 있다.

증거 수준: Science Advances peer-reviewed, ICML 2024

**메커니즘 5: 행동 신호 추적**

Zenko(Red Team): 비평의 가치는 품질이 아니라 **수용 의지**에 달렸다. Vaccaro et al. (Nature Human Behaviour 2024): 사용자가 AI 피드백에 노출되면 32.72%가 응답을 변경.

추적할 것:
- ~~"이 피드백이 도움됐나요?"~~ (만족도 — 노이즈)
- **"사용자가 계획을 수정했는가?"** (행동 — 신호)
- **"어떤 유형의 비평이 수정을 유발했는가?"** (비평 효과성 학습)

구현: signal-recorder에 `plan_revised_after_feedback` 이벤트 추가. 어떤 페르소나의 어떤 비평 유형이 실제 행동 변화를 만들었는지 축적.

증거 수준: Nature Human Behaviour meta-analysis, Zenko 17 case studies

### 하나의 반직관적 발견

Vaccaro et al. 메타분석: 인간-AI 조합이 **의사결정 과제**에서는 성과 손실, **콘텐츠 생성 과제**에서는 성과 향상.

Overture의 Rehearse는 콘텐츠 생성(비평, 가정 표면화)이다. 사용자가 결정한다.
→ **정확히 인간-AI 조합이 이득을 보는 아키텍처.**

### B의 구현 우선순위

| 우선순위 | 메커니즘 | 난이도 | Day 1 가능? | 가치 |
|---------|---------|-------|-----------|------|
| 1 | 진단적 증거 (가정 공격 필수화) | 낮음 (프롬프트 수정) | 예 | 높음 |
| 2 | 반복적 대화 | 중간 (UI 변경 필요) | 부분적 | 매우 높음 (Delphi r=0.95) |
| 3 | CoVe 검증 단계 | 중간 (추가 LLM 콜) | 예 | 높음 |
| 4 | 행동 신호 추적 | 낮음 (signal-recorder 확장) | 예 | 중간 (장기 학습) |
| 5 | 이질적 모델 앙상블 | 높음 (다중 API) | 아니오 | 높음 (9%p 향상) |

## 4. C: Backward-Forward Induction Loop

### 문제

세 번째 문서(judgment-data-foundation)의 결론: 참조 클래스 통계("73%가 실패")는 불투명하고, 행동 불가능하고, 생태학적 오류를 범한다. LLM 시뮬레이션으로 확률을 생성하면 McNamara 오류(정량화 불가를 정량화 후 진리로 취급)가 된다.

### 해법: 예측 대신 구조적 분석

참조 클래스 접근: "유사 결정의 73%가 실패했다"
→ 불투명, 비행동적, 업데이트 불가

Backward-Forward 접근: "이 결정이 성공하려면 7개 조건이 필요하다. 현재 2개가 위험 구간."
→ 투명, 행동적, 업데이트 가능, 개별 사례에 특화

### 이론적 기반

이 루프는 게임 이론, 의사결정 이론, 계산 이론에서 확립된 기초 위에 있다:

| 이론 | 기여 | 출처 |
|------|------|------|
| Backward Induction | 원하는 결과에서 역추적 → 가정 트리 생성 | Zermelo 1913, Selten 1965 |
| Forward Induction | 관찰된 행동에서 믿음 추론 → 가정 업데이트 | Kohlberg & Mertens 1986 |
| 강한 Δ-합리화 가능성 | Backward + Forward 결합의 형식적 기초 | Battigalli & Siniscalchi 2002 |
| 게임에서의 믿음 수정 | 루프의 형식화와 수렴 조건 | Stalnaker 1998 |
| CFR (Counterfactual Regret Minimization) | 이 루프의 계산적 구현 — 포커 AI 해결 | Zinkevich et al. 2007 |
| MCTS (Monte Carlo Tree Search) | 샘플링 기반 forward-backward — AlphaGo | Kocsis & Szepesvari 2006 |
| 사전부검 | Backward induction의 경험적 검증 — 리스크 식별 +30% | Klein 1998 |
| 가정 기반 계획 | Forward induction의 운용화 | Dewar et al. 1993, RAND |
| 더블루프 학습 | Backward-Forward의 조직 이론 버전 | Argyris & Schon 1978 |

### Overture 4R로의 매핑

```
REFRAME (Backward Induction)
─────────────────────────────
입력: 사용자의 결정/질문
처리: 원하는 결과에서 역추적
출력: 가정 트리 A = {a₁, a₂, ..., aₙ}
      각 가정에 사전 확률, 중요도, 무효화 조건

  + A의 Layer 1~4가 여기서 작동:
    - 도메인 기저율을 외부 관점으로 제시 (L1)
    - 프로세스-구조 리스크 자동 감지 (L2)
    - 구조 조건부 편향 경고 (L3)
    - 맥락-접근법 정합성 체크 (L4)

         │
         ▼

RECAST (다중 관점 Backward)
─────────────────────────────
입력: 가정 트리 + 실행 설계
처리: 각 페르소나가 다른 "성공" 정의에서 역추적
      CFO의 가정 트리 ≠ CTO의 가정 트리 ≠ 고객의 가정 트리
출력: 관점별 가정 트리의 교집합과 차집합
      → 누가 어떤 가정을 가장 위협하는지 매핑

         │
         ▼

REHEARSE (Forward Induction)
─────────────────────────────
입력: 통합 가정 트리 + 페르소나 프로필
처리: 이해관계자 반응 = 신호 관찰
      각 가정에 대해 베이지안 업데이트

  + B의 메커니즘이 여기서 작동:
    - 진단적 증거: 가정별 구체적 공격 (B1)
    - CoVe: 비평의 독립 검증 (B2)
    - 반복 대화: 사용자 반박 → 페르소나 재응답 (B3)
    - 행동 신호 추적 (B5)

출력: 업데이트된 가정 확률
      위협받는 가정 목록 T = {aᵢ : p(aᵢ) < 임계치}

         │
         ▼

REFINE (반복 / 수렴)
─────────────────────────────
입력: 위협 가정 + 업데이트된 가정 트리
처리: 위협 가정에 대해 재(再) Backward Induction
      "aᵢ가 거짓이면 결정은 어떻게 되는가?"
      적응 전략, 대안 경로, 또는 "불가" 판별
출력:
  수렴한 경우:
    - 최종 가정 트리 (각 가정의 상태: 검증됨/위협/불확실)
    - 적응 전략 (위협 가정별)
    - 모니터링 계획 (각 가정의 신호 포인트 + 30/90/180일 체크)

  수렴하지 않은 경우:
    - "해소되지 않는 핵심 긴장" 목록
    - 추가 정보가 필요한 영역
    - "이 결정은 현재 정보로는 판단 불가" 진단

수렴 기준:
  - 모든 가정이 임계치 이상 → 결정 유지
  - 적응 전략이 위협 가정을 커버 → 조건부 유지
  - 핵심 가정이 임계치 미만 + 적응 불가 → 결정 재고
  - 3라운드 이내 믿음이 안정화 → 수렴 (과도한 반복 방지)
```

### 왜 이게 참조 클래스보다 나은가

| 차원 | 참조 클래스 ("73% 실패") | BF Loop ("7개 조건, 2개 위험") |
|------|----------------------|---------------------------|
| 투명성 | 숫자의 출처 불투명 | 모든 가정에 출처와 근거 |
| 행동 가능성 | 핸들 없음 | 어떤 가정을 작업할지 명확 |
| 정직성 | 불확실성이 숫자 뒤에 숨음 | 어디서 확신하고 어디서 추측하는지 보임 |
| 업데이트 | 정적 | 새 신호 → 특정 가정만 업데이트 |
| 특이성 | 집단 평균 | 이 결정에 특화된 분석 |
| 오류 유형 | 생태학적 오류 (집단→개인) | 가정 누락 가능 (완화 가능) |

### C의 구현 우선순위

| 우선순위 | 요소 | 난이도 | Day 1 가능? | 가치 |
|---------|------|-------|-----------|------|
| 1 | Reframe에서 가정 트리 명시적 생성 | 중간 (프롬프트 재설계) | 예 | 매우 높음 |
| 2 | 가정별 사전 확률 + 무효화 조건 | 낮음 (출력 스키마 확장) | 예 | 높음 |
| 3 | Rehearse→Refine 가정 업데이트 루프 | 중간 (상태 관리) | 예 | 매우 높음 |
| 4 | 수렴 기준 형식화 | 중간 (로직) | 예 | 중간 |
| 5 | 모니터링 계획 출력 (30/90/180일) | 낮음 (출력 추가) | 예 | 높음 (가정 추적) |

## 5. 통합 구현 로드맵

### Phase 1: 프롬프트 레벨 통합 (Day 1 — 코드 변경 최소)

A, B, C 모두 프롬프트 수정만으로 시작할 수 있는 요소가 있다:

**Reframe 프롬프트 개정:**
- 기존: hidden_assumptions 발견 + reframed_question 생성
- 추가: **가정 트리 명시적 생성** (backward induction)
  - 각 가정에: 사전 확률 추정, 중요도, 무효화 조건, 모니터링 신호
- 추가: **프로세스-구조 리스크 자동 감지** (A Layer 2)
  - Edict 전술 감지, 단일 옵션 감지, T-T 미스매치 감지
- 추가: **구조 조건부 편향 경고** (A Layer 3)
  - 이 결정의 구조에서 가장 활성화될 편향 1~2개만

**Rehearse 프롬프트 개정:**
- 기존: 페르소나별 overall_reaction + classified_risks
- 변경: **진단적 증거 필수화** (B 메커니즘 1)
  - 각 비평에 "공격하는 가정", "가정이 틀렸을 때의 구체적 결과" 필수
  - `untested_assumptions`를 1순위 출력으로 승격
- 추가: **가정 확률 업데이트 출력** (C Forward 단계)
  - "이 페르소나의 피드백에 비추어, 다음 가정의 확률이 변경됩니다: ..."
- 추가: **도메인 기저율 주입** (A Layer 1)
  - 관련 기저율을 페르소나 프롬프트에 근거 데이터로 제공

**Refine 프롬프트 개정:**
- 기존: 변경 사항 추적 + 수렴 확인
- 추가: **가정 트리 상태 업데이트** (C 반복)
  - 각 가정: 검증됨 / 위협 / 불확실 / 무효화
- 추가: **모니터링 계획** (C 출력)
  - 위협 가정별 30/90/180일 체크 포인트

### Phase 2: 구조적 확장 (1~2개월)

- **반복적 대화 UI** — 사용자가 비평에 반박하고 페르소나가 재응답 (B 메커니즘 3)
- **CoVe 검증 파이프라인** — 비평 생성 후 독립 검증 LLM 콜 (B 메커니즘 2)
- **가정 추적 대시보드** — 프로젝트별 가정 트리 시각화, 시간에 따른 확률 변화
- **행동 신호 확장** — plan_revised_after_feedback 이벤트 (B 메커니즘 5)

### Phase 3: 데이터 축적 (6개월+)

- **개인 보정 곡선** — 100+개 예측 축적 후 보정 피드백 (A Layer 5)
- **비평 효과성 학습** — 어떤 유형의 비평이 실제 계획 수정을 유발하는지 패턴
- **이질적 모델 앙상블** — 다중 LLM 제공자 통합 (B 메커니즘 4)

## 7. 이것이 "LLM한테 잘 물어보기"와 어떻게 다른가

문서 2의 핵심 경고: *"A와 C가 연결되지 않으면 'LLM한테 잘 물어보기'와 본질적으로 구별되지 않는다."*

BF Loop 통합 후의 차이:

| 차원 | "LLM한테 잘 물어보기" | Overture + BF Loop |
|------|-------------------|-------------------|
| 구조 | 단일 패스 질문-답변 | 반복적 backward-forward 루프 |
| 근거 | LLM의 훈련 데이터 | 공개 연구 기저율 + 구조적 편향 데이터 |
| 비평 | 요청하면 제공 | 진단적 증거 + 독립 검증 + 반복 대화 |
| 출력 | 텍스트 답변 | 가정 지도 + 모니터링 계획 + 적응 전략 |
| 학습 | 없음 | 개인 보정 곡선 + 비평 효과성 축적 |
| 수렴 | 없음 (사용자가 만족하면 끝) | 형식적 수렴 기준 (가정 안정화) |
| 정직성 | LLM이 자신 있게 답함 (84.3% 과신) | 불확실성 명시 + "모르겠다" 허용 |

**한 문장 차이:**
LLM한테 잘 물어보기는 "좋은 답변"을 준다.
Overture + BF Loop은 "당신이 아직 묻지 않은 질문"을 찾아준다.
