# Reframing Strategies

## 1. Challenge Existence

**When:** User doubts the fundamental premise. Most assumptions are doubtful.

Ask "what happens if we just... don't do this?" before anything else. The problem might not need solving — or there's a completely different problem underneath.

**Example:**
- Original: "Build an AI onboarding chatbot for our product"
- Reframed: "Our onboarding completion rate is 34%. Is the problem that users need more guidance, or that the product is too complex? A chatbot treats the symptom."

## 2. Narrow Scope

**When:** Direction is clear but scope is paralyzing. User knows where to go but not how to start.

Find the smallest meaningful experiment. What's the ONE thing to test first? Note: The main reframed question stays broad — the narrowing happens in the hidden questions and suggested first steps.

**Example:**
- Original: "Develop a go-to-market strategy for enterprise"
- Reframed: "Which 3 enterprise prospects can we close in the next 30 days without any product changes, to validate our enterprise value prop?"

## 3. Diagnose Root Cause

**When:** The situation is urgent (fire-fighting mode). Risk of treating symptoms instead of causes.

Force the "why now?" question. Separate immediate response from structural fix.

**Example:**
- Original: "Our churn rate spiked this month — we need a retention campaign"
- Reframed: "Is this churn spike caused by a product issue, a competitive shift, or a cohort quality problem? Each requires a completely different response."

## 4. Redirect Angle

**When:** Multiple stakeholders with competing interests, or the problem needs a fundamentally different perspective.

Look at the same situation through a different stakeholder's eyes. "What if this problem were owned by a different department?"

**Example:**
- Original: "Set pricing for our B2B SaaS product"
- Reframed: "Pricing is an output of value perception, not an input. How do we design the customer's perception of value so that price becomes obvious?"

## Selection logic

```
Mostly confident assumptions     → Narrow Scope or Redirect Angle
Mixed (some doubtful)            → depends on pattern:
  ├─ "Why" is unclear            → Challenge Existence
  ├─ "How" is unclear            → Narrow Scope
  ├─ Urgent/crisis               → Diagnose Root Cause
  └─ Stakeholder conflict        → Redirect Angle
Mostly doubtful assumptions      → Challenge Existence
```

## What a bad reframe looks like

A bad reframe just adds words or vague qualifiers:
- Original: "We need to expand into SEA"
- Bad reframe: "We need to strategically and comprehensively expand into SEA with a clear plan" — this changes nothing
- Bad reframe: "How can we optimize our SEA expansion strategy?" — this is the same question in fancier words

A good reframe changes what you're LOOKING FOR, not how you phrase the search.
