# Overture 브랜딩: 오케스트라 메타포 정교화

> 기능은 철저히 기능 단위로 설계. 이 문서는 **브랜딩/래핑** 레이어에 대한 정리.

---

## 1. 왜 음악인가 — 프레임 선택의 근거

### 검토한 두 프레임

| | 오케스트라 (Overture) | 법정 (Court) |
|---|---|---|
| **강점** | 가볍고 창의적, "agent 가기 전"이 이름에 내장, 반복이 자연스러움 | 기능 매핑이 1:1, 역할이 즉시 이해됨, 모두가 아는 구조 |
| **약점** | 매핑이 느슨할 수 있음, 의식적으로 심어야 작동 | 무겁고 적대적, "피고인석" 느낌, 끝없는 재판 = 카프카 |

### 결론: 오케스트라 유지, 법정의 구조적 명확성을 차용

- **법정의 핵심 장점**은 역할이 명확하다는 것 → 오케스트라에서도 악기별 고정 성격으로 해결 가능
- 법정 프레임의 문제: "누군가 틀렸다"는 전제, 방어적 태도 유발, 무한 법정 = 지옥 메타포
- 음악 프레임의 톤: 리허설은 **더 잘하려고** 하는 것. 반복해도 자연스러움 (오케스트라는 수십 번 리허설함)

## 2. 전체 흐름: Movement 구조

4R 파이프라인을 악장(Movement)으로 래핑:

```
Movement I.   Reading the Score     → Reframe
              악보를 읽고 해석한다. 숨겨진 전제를 발견하고 질문을 재정의.

Movement II.  Orchestration         → Recast
              편곡한다. AI와 사람의 역할을 분배하고 실행을 설계.

Movement III. Rehearsal             → Rehearse + Refine
              리허설. 연주자들(페르소나)이 검증하고, 수렴할 때까지 반복.

Movement IV.  Curtain Call          → 결과 출력
              결과물 + DQ Score + Program Note.
```

### Movement III의 반복 구조: Rehearsal Mark

실제 악보에서 리허설 마크(A, B, C, D...)는 구간을 표시하고, "Rehearsal Mark B로 돌아가서 다시"라고 지시함.

- Refine loop의 각 iteration = Rehearsal Mark A → B → C → ...
- 자연스러운 반복 내러티브: "아직 맞지 않으니 B로 돌아가자"
- 법정의 1심/2심/3심 대비 장점: 법정은 "이전 판결이 잘못됐을 때"만 올라가지만, 리허설은 "더 좋게 만들기 위해" 반복

## 3. 페르소나 = 연주자 (Ensemble Branding)

### 핵심 원칙

> 내부 로직은 기능적 분류(Adversarial, Analytical, Empathetic, Integrative).
> 악기 이름은 그 위에 입히는 **스킨**. 구조를 메타포에 종속시키지 않는다.

### 악기군(Section) → 기능 분류 매핑

```
┌─────────────────────────────────────────────────────────────────┐
│  Brass Section (금관)          ←  Adversarial                   │
│  ─────────────────────────────────────────────────────────────  │
│  강하고 직접적. 반론, 스트레스 테스트, 도발적 질문.              │
│                                                                 │
│  🎺 Trumpet    — 핵심 반론. 가장 날카롭고 직접적인 도전         │
│  🎵 Trombone   — 경쟁/대안 시나리오. 다른 길을 보여줌           │
│  📯 French Horn — 숨겨진 리스크 발굴. 간과하기 쉬운 구조적 약점 │
├─────────────────────────────────────────────────────────────────┤
│  Strings Section (현악)        ←  Analytical                    │
│  ─────────────────────────────────────────────────────────────  │
│  정교하고 체계적. 데이터, 구조, 논리 분석.                      │
│                                                                 │
│  🎻 First Violin  — 핵심 논리 구조 분석. 주선율을 이끔          │
│  🎻 Second Violin — 데이터/증거 기반 검증                       │
│  🎵 Viola         — 실행 가능성/현실성 체크                     │
│  🎵 Cello         — 장기 관점. 시간 축에서의 리스크와 기회      │
│  🎵 Double Bass   — 기본 전제/토대 검증. 가장 근본적인 질문     │
├─────────────────────────────────────────────────────────────────┤
│  Woodwinds Section (목관)      ←  Empathetic                    │
│  ─────────────────────────────────────────────────────────────  │
│  섬세하고 다채로운 색채. 사용자, 고객, 이해관계자 시선.         │
│                                                                 │
│  🎵 Flute    — 최종 사용자/고객 관점                            │
│  🎵 Oboe     — 내부 팀/실무자 관점                              │
│  🎵 Clarinet — 시장/경쟁 환경 관점                              │
│  🎵 Bassoon  — 규제/법률/컴플라이언스 관점                      │
├─────────────────────────────────────────────────────────────────┤
│  Percussion Section (타악)     ←  Integrative                   │
│  ─────────────────────────────────────────────────────────────  │
│  리듬과 구조를 잡는다. 종합, 밸런스, 의사결정 프레이밍.         │
│                                                                 │
│  🥁 Timpani    — 전체 균형/종합 판단                            │
│  🎵 Xylophone  — 우선순위 정리. 무엇이 먼저인가                │
│  🎵 Triangle   — 미묘한 신호 포착. 놓치기 쉬운 작은 단서       │
└─────────────────────────────────────────────────────────────────┘
```

### 일관성 원칙

**악기 = 고정된 성격**. 매번 랜덤이면 안 된다.

- Trumpet은 항상 가장 날카로운 반론
- Cello는 항상 장기적 관점
- Flute는 항상 사용자/고객 시선

이 일관성이 있어야:
1. 사용자가 학습함 — "아 Trumpet이 뭐라 했는지 봐야지"
2. 애착이 생김 — 자주 쓰는 악기 조합이 "내 앙상블"이 됨
3. 대화가 됨 — "Trumpet이 반대했는데 설득력 있었어" (팀 공유 시)

## 4. Smart Ensemble: 자동 편성 추천

### 개념

과제 성격과 과거 데이터를 기반으로 **최적의 악기 조합을 자동 추천**.

```
투자 의사결정 →  🎺 Trumpet + 📯 French Horn + 🎵 Cello + 🎻 Second Violin + 🥁 Timpani
                 (반론 + 숨은 리스크 + 장기 관점 + 데이터 검증 + 종합)

브랜딩 전략   →  🎵 Flute + 🎵 Clarinet + 🎵 Oboe + 🎺 Trumpet + 🥁 Timpani
                 (고객 + 시장 + 내부 팀 + 반론 + 종합)

채용 결정     →  🎵 Oboe + 🎵 Viola + 🎵 Flute + 🎵 Triangle + 🥁 Timpani
                 (내부 팀 + 현실성 + 후보자 관점 + 미묘한 신호 + 종합)
```

### 성장 내러티브

데이터가 쌓이면 추천이 정교해짐:

- **초기**: "이런 유형의 결정에는 보통 이 5인 앙상블"
- **중기**: "당신의 과거 DQ 데이터 기반, 이 조합이 가장 효과적이었음"
- **장기**: "이 조합에서 French Horn을 빼면 DQ가 12% 떨어지는 경향 — 유지 추천"

## 5. 규모의 브랜딩: 편성 크기

결정의 복잡도/중요도에 따라 편성 규모가 달라짐:

```
Solo           (1인)    — 빠른 체크. "이거 괜찮아?"
Duet           (2인)    — 찬반 대립. 핵심 트레이드오프 검토
Quartet        (4인)    — 표준 검토. 대부분의 결정에 적합
Chamber        (6-8인)  — 심층 검토. 중요한 전략적 판단
Full Orchestra (10+인)  — 최고 강도. 조직 차원의 대형 의사결정
```

추천 시 표현 예시:
- "이 결정에는 **현악 4중주**면 충분합니다"
- "이건 **풀 오케스트라**가 필요한 수준입니다"

사용자가 성장하면서 자연스럽게 더 큰 편성을 다루게 됨 → **제품 성장 = 음악적 성장** 내러티브.

## 6. 출력물: Program Note

콘서트 프로그램 노트 형식으로 결과 래핑:

```
┌─────────────────────────────────────────────────┐
│  PROGRAM NOTE                                   │
│  ─────────────────────────────────────────────── │
│                                                  │
│  Title: [결정 제목]                              │
│  Ensemble: Chamber (6인)                         │
│  Movements: I → II → III (×3) → IV              │
│  Performance Score: 78 / 100                     │
│                                                  │
│  ── Synopsis ──                                  │
│  [1-3줄 요약]                                    │
│                                                  │
│  ── Ensemble Notes ──                            │
│  🎺 Trumpet: "핵심 반론 — ..."                   │
│  🎵 Cello: "장기적으로 보면 ..."                 │
│  🎵 Flute: "사용자 입장에서 ..."                 │
│  ...                                             │
│                                                  │
│  ── Conductor's Remark ──                        │
│  [종합 판단 + 추천 행동]                         │
│                                                  │
│  ── Next Movement ──                             │
│  [다음 단계 제안]                                │
│                                                  │
└─────────────────────────────────────────────────┘
```

**Conductor's Remark** = 지휘자의 총평. DQ Score와 함께 가장 핵심적인 종합 판단.

## 7. 특수 역할: Concertmaster (악장)

이미 존재하는 Concertmaster 컨셉과 연결:

- Concertmaster = 수석 바이올리니스트 = **앙상블 전체를 조율하는 역할**
- 개별 악기가 각자의 관점을 내놓은 후, Concertmaster가 수렴/조율
- Refine loop에서 "아직 안 맞으니 다시"를 판단하는 주체
- 최종 Program Note의 Conductor's Remark를 작성하는 주체

Conductor(지휘자) vs Concertmaster(악장) 구분:
- **Conductor** = 시스템(Overture 엔진 자체). 전체 흐름을 지휘
- **Concertmaster** = 앙상블 내 수석. 연주자들 사이에서 조율

## 8. 성장 내러티브: 점점 풍부해지는 관현악

### 제품 진화 = 오케스트라 확장

```
v1.0  기본 앙상블    — 핵심 악기 8-10종. 대부분의 판단에 충분
v2.0  확장 편성      — 특수 악기 추가 (하프, 첼레스타, 색소폰...)
                       → 특수 도메인 전문가 (의료, 법률, 기술 심층)
v3.0  객원 연주자    — 외부 데이터/API 연동 페르소나
                       → 실시간 시장 데이터, 경쟁사 분석 등
v4.0  작곡가 모드    — 사용자가 직접 악기(페르소나) 정의
                       → 커스텀 앙상블, 팀별 특화 편성
```

### 사용자 개인 성장

```
입문   "추천 앙상블 그대로 쓰기"
중급   "악기 조합을 직접 조정" — Trumpet 빼고 French Horn 추가
고급   "나만의 앙상블 저장" — 자주 쓰는 조합을 프리셋으로
마스터 "앙상블 공유" — 팀/커뮤니티에 내 편성 공유
```

## 9. 톤 가이드라인

### 쓸 것
- "앙상블을 구성합니다" (페르소나 생성 시)
- "리허설을 시작합니다" (검증 단계 진입 시)
- "Rehearsal Mark B로 돌아갑니다" (Refine loop 시)
- "이 결정에는 현악 4중주면 충분합니다" (규모 추천 시)
- "Trumpet이 강하게 반대하고 있습니다" (핵심 반론 표시)

### 쓰지 않을 것
- 과도한 음악 전문 용어 (sforzando, pizzicato, rubato...)
- 악기를 아는 사람만 이해할 수 있는 비유
- "연주" 자체에 대한 메타포 (Overture는 연주가 아니라 리허설)
- 강제적 음악 교육 ("이 악기는 오케스트라에서..." 같은 설명)

### 원칙
> 메타포는 **이해를 돕는 도구**이지 **학습해야 하는 지식**이 아니다.
> "Trumpet이 반대합니다"는 직관적. "Trumpet의 음역대 특성상..."은 과잉.
> 악기 이름을 몰라도 아이콘과 한 줄 설명으로 역할이 보여야 한다.

## 10. 기존 용어 규칙과의 정합

| 기존 규칙 | 이 문서에서의 적용 |
|---|---|
| "Overture = 단일 악장" → 1악장/2악장 금지 | Movement I~IV는 "악장 번호"가 아니라 흐름 단계. 기존 규칙의 취지(Overture 자체가 하나의 곡)와 충돌하지 않음 — Movement는 그 하나의 곡 안의 구조 |
| 컨설팅 용어 금지 | 유지. "Decision Brief" → "Program Note", "Risk Assessment" → "French Horn의 발견" |
| Concertmaster = 악장 | 유지. 수석 바이올리니스트 역할로 일관 |

## 부록: 법정 메타포에서 차용할 어휘

법정 프레임 자체는 채택하지 않지만, 일부 어휘는 리허설 맥락에서 자연스럽게 차용 가능:

- "반론" (objection) → Brass section이 제기
- "증거" (evidence) → Strings section이 제시
- "심리" (hearing) → Rehearsal의 각 라운드
- "최종 변론" (closing argument) → Conductor's Remark

이 어휘들은 음악 프레임 안에서 **리허설 중 토론**의 맥락으로 사용. 전체 프레임을 법정으로 전환하지 않음.
